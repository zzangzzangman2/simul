# SIMUL 작업 필수 규칙

이 저장소에서 작업하는 사람과 에이전트는 아래 조건을 임의로 제거하거나 약화하면 안 됩니다. 상세 배경과 구현 구조는 `PROJECT_GUIDE.md`를 먼저 읽습니다.

1. 제품 기준 화면은 **모바일 세로 모드 390×844px**이다.
2. 최소 360px 너비에서 가로 스크롤이 생기면 안 된다.
3. PC 브라우저에서도 폭 430px 이하의 같은 세로형 앱 프레임을 가운데 표시한다. 넓은 데스크톱 대시보드로 확장하지 않는다.
4. 앱은 `처음하기`·`이어하기` 타이틀로 시작한다. 처음하기를 고르면 `1999.12.31 23:57` 가족 스토리 프롤로그가 열리고, 가족 대화 뒤에 주인공과 투자회사 이름을 정한다.
5. 로컬 저장은 최대 5슬롯이며 기존 단일 저장은 1번 슬롯으로 보존한다. 이어하기에서 삭제할 수 있고, 날짜가 하루 넘어갈 때마다 활성 슬롯에 자동저장하며 상단 게임 메뉴에서 수동저장한다. 주식시장을 화면 뒤로가기나 시스템 Back으로 닫을 때도 현재 장중 시각의 저장 성공을 확인한 뒤 한 번만 화면을 닫는다.
6. 신규 게임의 첫 플레이 날짜는 **2000년 1월 2일 일요일 08:00**이다. 날짜가 바뀌면 항상 다음 날 08:00으로 이동하며 기존 저장의 날짜는 보존한다.
7. 신규 플레이는 **초기자본 0원, 창립 인원 1명**으로 시작하고 누적 종잣돈 1만원을 직접 벌어 첫 주문 권한을 연다. 가족 조언자는 정식 직원 수에 포함하지 않으며 일거리·거래·하루 진행을 장부에 남긴다.
8. 거래 가능한 회사는 가상기업만 사용한다. 출발 30개 기업의 이름·업종·제품은 고정하고 가격·사건·신규상장·분사·상장폐지는 월드시드로 결정한다.
9. 일별 시장가격과 장중 경로는 저장된 월드시드와 공개된 사건으로 생성하며, 주말·휴장일에는 직전 종가를 유지한다.
10. 가족 프롤로그 완료 후에는 주식 목록이 아니라 세로형 아파트 허브의 작은방이 먼저 보이고, 작은방 CRT 컴퓨터를 눌러야 주식시장이 열린다.
11. 종목은 1열 목록, 거래는 바텀시트를 유지하며 하단의 독립된 `1시간 보내기`·`하루 보내기` 버튼은 명확히 보여야 한다. `하루 보내기`는 당일 20:00 저장과 전날 신문 생성을 끝낸 뒤 숨은 시나리오가 고정된 다음 날 08:00 상태를 먼저 저장하고 신문을 연다.
12. 기능 변경 후 타이틀, 5슬롯, 프롤로그, 3공간, 거래·AUM, v14 마이그레이션, 30개 고정 기업, 수천 사건, 업종 금지 조합, 신문 비누설, 보고서 단회 구매, 장중 속보와 가로 넘침을 검증한다.
13. `DATA_SOURCES.md`의 출처·라이선스 주의사항을 삭제하지 않는다.
14. `flutter analyze`, `flutter test`, `npm run build:release`, `npm test`가 통과한 상태만 `main`에 푸시한다.
15. 주가창 기본 화면은 가상기업만 표시하고 `미래시장`·`도전시장`·`신규·분사`·`관심`으로 나눠 본다.
16. 장중 틱은 게임용 자동 생성이라고 표시한다. 08:00~08:59는 이전 종가로 고정하고 09:00부터 1분봉을 만들며, 캠페인 시대(2000~2010년)에 맞춰 14:50~15:00 종가 동시호가 뒤 20:00까지 추가 거래·분봉 없이 같은 가격을 유지한다.
17. Gemini는 전날 공개된 시장 사실의 기사 문장만 생성한다. 오늘의 숨은 시나리오, 보고서 힌트, 영향률, 성공 여부와 미래 가격을 입력하지 않으며 가격·주문·기업행동을 AI가 바꾸게 하지 않는다.
18. 실제 모바일 확인은 개발 서버를 `0.0.0.0:8000`에 열고 PC와 같은 Wi-Fi의 휴대폰에서 `http://<PC 내부 IP>:8000`으로 접속해 수행한다. `localhost` 확인만으로 실기기 검증을 대신하지 않는다.
19. 아파트 허브는 실제 배경의 물건을 누르는 세로형 3공간으로 유지한다. 작은방 CRT·서류함은 시장·고급 포트폴리오로, 거실 안건 편지·가족 공간은 안건함·가족/조직으로, 부엌 유선전화는 일거리로 연결한다. 모든 핫스폿은 배경에 그려진 실제 오브젝트의 위치와 일치해야 한다.
20. 사용자가 정한 회사 이름을 임의의 제품명이나 접미사로 덮지 않는다. 방·시장·사람들·서류함의 주 표시에는 저장된 이름을 그대로 사용한다.
21. 이름 입력 중에는 브라우저의 고정 높이 Flutter 호스트로 배경·상단·대화창의 전체 레이아웃을 유지한다. 모바일 키보드 높이는 이름 패널에만 한 번 반영하고 360×800에서 입력창과 완료 버튼이 키보드 위에 남는지 검증한다.
