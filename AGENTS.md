# SIMUL 작업 필수 규칙

이 저장소에서 작업하는 사람과 에이전트는 아래 조건을 임의로 제거하거나 약화하면 안 됩니다. 상세 배경과 구현 구조는 `PROJECT_GUIDE.md`를 먼저 읽습니다.

1. 제품 기준 화면은 **모바일 세로 모드 390×844px**이다.
2. 최소 360px 너비에서 가로 스크롤이 생기면 안 된다.
3. PC 브라우저에서도 폭 430px 이하의 같은 세로형 앱 프레임을 가운데 표시한다. 넓은 데스크톱 대시보드로 확장하지 않는다.
4. 첫 방문자는 `1999.12.31 23:57` 가족 스토리 프롤로그부터 시작하고, 가족 대화 뒤에 주인공과 가족 투자연구소 이름을 정한다.
5. 회사 이름과 게임 진행상태는 같은 브라우저의 로컬 저장소에 자동 저장한다.
6. 게임 시작일은 **2000년 1월 1일**이며, 미래 시점의 정보를 미리 보여주면 안 된다.
7. 신규 플레이는 **현금 0원, 가족 투자연구소**로 시작하며 일거리 수입과 하루 진행을 장부에 남긴다.
8. 실제 시장 데이터와 게임 밸런싱용 가상 수치·딜을 화면과 문서에서 구분한다.
9. 일별 시장가격은 해당 날짜까지 공개된 실제 거래일 데이터만 사용하고, 주말·휴장일에는 직전 종가를 유지한다.
10. 가족 프롤로그 완료 후에는 주식 목록이 아니라 내 방이 먼저 보이고, 방의 CRT 컴퓨터를 눌러야 주식시장이 열린다.
11. 종목은 1열 목록, 거래는 바텀시트를 유지하며 하단 `1시간 뒤`·`오늘 신문` 버튼의 글자는 명확히 보여야 한다.
12. 기능 변경 후 모바일 프롤로그, 연구소 이름 저장, 방 시작, 컴퓨터 진입, 실제 매수·매도와 AUM, 시간 진행·오늘 신문, 거래 바텀시트, React v3 저장 가져오기와 가로 넘침을 검증한다.
13. `DATA_SOURCES.md`의 출처·라이선스 주의사항을 삭제하지 않는다.
14. `flutter analyze`, `flutter test`, `npm run build:release`, `npm test`가 통과한 상태만 `main`에 푸시한다.
15. 주가창 기본 화면은 국내 시장이며 KOSPI·KOSDAQ을 나눠 본다. 기존 미국·일본 데이터는 `해외` 탭에 보존한다.
16. 장중 틱은 게임용 자동 생성이라고 표시하고, 거래일의 마지막 틱은 실제 일별 종가와 정확히 일치해야 한다.
17. Gemini는 기사·대화 서사만 생성한다. 인증은 서버에만 두고, AI 실패 시 결정론적 신문으로 폴백하며 실제 종가·주문 계산을 AI가 직접 바꾸게 하지 않는다.
18. 실제 모바일 확인은 개발 서버를 `0.0.0.0:8000`에 열고 PC와 같은 Wi-Fi의 휴대폰에서 `http://<PC 내부 IP>:8000`으로 접속해 수행한다. `localhost` 확인만으로 실기기 검증을 대신하지 않는다.
19. 작은방은 메뉴 카드 목록이 아니라 실제 방 배경의 물건을 누르는 허브로 유지한다. CRT는 시장, 책상 위 편지는 안건함, 책상 서랍은 포트폴리오·원장 전체 화면으로 연결한다.
20. 사용자가 정한 회사 이름을 임의의 제품명이나 접미사로 덮지 않는다. 방·시장·사람들·서류함의 주 표시에는 저장된 이름을 그대로 사용한다.
21. 이름 입력 중에는 배경과 대화창을 동시에 축소하지 않는다. 모바일 키보드 높이를 한 번만 반영하고 360×800에서 입력창과 완료 버튼이 키보드 위에 남는지 검증한다.
