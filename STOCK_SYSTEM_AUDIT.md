# 주식 시스템 현재 감사 결과

최종 갱신: 2026-07-22

대상: Flutter 게임, Flutter Web `/play/`, 웹 배포 셸, 시장 데이터 스키마 `4`, 게임 저장 스키마 `v13`.

## 결론

주문 화면과 엔진의 시세·날짜·분 검증이 하나의 권위 경로로 통합되어 있습니다. 신규 게임은 0원으로 시작하며 직접 번 누적 종잣돈과 보호자 주문 권한을 통과해야 거래할 수 있습니다. 차트 최소 단위는 1분이고, 시장 화면이 열린 동안에만 현실 1초마다 게임 1분이 진행됩니다.

## 데이터와 시세

- 실제 일별 종가 원장을 게임 가격의 기준으로 사용합니다.
- 장중 가격은 이전 종가에서 해당 거래일 종가로 수렴하는 결정론적 1분봉입니다.
- 화면 표시가, 주문 견적, 체결가는 같은 종목·날짜·시장 분으로 다시 계산됩니다.
- 오래된 견적 날짜나 분, 비정상 수량, 잔액 부족, 보유량 초과, 휴장 주문은 상태를 바꾸지 않고 거절합니다.
- 분할과 배당은 명시적 기업행동으로 보유 수량·현금·원장을 한 번만 갱신합니다.
- USD·JPY 주문과 원화 AUM 합산은 환율 원장이 연결되기 전까지 잠급니다.

## 회계

- 매수: 체결대금과 수수료를 현금에서 차감하고 수량·총원가를 갱신합니다.
- 매도: 평균원가 기준으로 총원가를 비례 차감하고 순매도대금과 실현손익을 원장에 기록합니다.
- AUM: 현금과 현재 평가 가능한 원화 포지션만 합산합니다.
- 월 이력: 현금·포트폴리오 원가·실현손익·평판·게임 기준지수를 최대 132개월 저장합니다.
- 월 경제: 급여, 사무실 임대료, 직원 리서치 수입, 예수금 이자, 펀드 보수, 운영 회사 수입을 날짜 순서대로 반영합니다.

## 시장 시간·달력

- 기본/최소 봉: 1분.
- 실제 타이머: 주식시장 화면에서만 현실 1초당 게임 1분.
- 화면이 닫히거나 주문·저장 중이면 자동 타이머 정지.
- 시스템 Back은 현재 시장 시각 저장을 완료한 뒤 화면을 닫습니다.
- 거래일 판단은 주말·고정 휴일·실제 국내 종가 공백에서 추출한 휴장일을 공용 달력에서 처리합니다.

## 저장 방어

- React 웹 `v3`과 이전 Flutter 저장을 `v13`으로 보존 마이그레이션합니다.
- 마이그레이션은 0원 잔액을 그대로 보존하고 자본을 임의 지급하지 않습니다.
- 거래할 수 없는 레거시 해외 포지션은 기록된 원가를 원화 현금으로 한 번 복구합니다.
- 기본 저장과 최근 정상 백업을 분리하며 손상 원문을 덮어쓰지 않습니다.
- 대체역사 저장의 가격 누락은 안전한 결정론적 가격으로 폴백합니다.
