# MILLENNIUM CAPITAL 프로젝트 인수인계 문서

이 문서는 다른 PC, 다른 개발 환경, 다른 에이전트가 저장소를 처음 열어도 현재 구현을 이해하고 이어서 작업할 수 있도록 작성한 기준 문서입니다.

## 1. 제품 한 줄 정의

2000년 1월 1일부터 실제 역사적 시장 흐름을 따라 상장회사에 투자하고, 팀과 평판을 키우며, 실제 M&A에서 착안한 인수 기회를 실행하는 모바일 세로형 투자회사 경영 시뮬레이션입니다.

## 2. 절대 유지해야 하는 제품 요구사항

### 화면과 조작

- 디자인 기준 기기는 모바일 세로 모드 `390×844px`입니다.
- 최소 지원 너비는 `360px`입니다.
- 모바일에서 가로 스크롤이 생기면 실패로 봅니다.
- 주요 버튼은 손가락으로 누르기 쉽도록 최소 44px 높이를 유지합니다.
- 하단의 `하루 보내기` 버튼은 모바일 안전영역을 침범하지 않아야 하며 밝은 버튼 위 글자는 검정색으로 선명하게 보여야 합니다.
- PC 브라우저에서도 전체 플레이와 테스트가 가능해야 합니다.
- PC에서도 폭 430px 이하의 동일한 세로형 앱 프레임을 가운데 표시합니다. 넓은 데스크톱 대시보드로 확장하지 않습니다.
- 회사 설립 직후에는 주식 목록이 아니라 2000년의 작은 원룸 사무실에서 시작합니다.
- 주식시장은 방 안의 CRT 컴퓨터를 눌러야 열립니다.
- 유선전화는 인수 제안, 서류함은 포트폴리오로 연결되는 방 기반 허브를 유지합니다.
- 종목은 모바일 1열 목록으로 표시하고, 종목을 누르면 거래 바텀시트가 즉시 열려야 합니다.

### 첫 진입

- 첫 방문 시 회사 이름 입력 화면이 가장 먼저 보여야 합니다.
- 공백만 입력한 이름으로 시작할 수 없습니다.
- 회사 이름은 최대 24자입니다.
- 설립 후 회사 이름은 상단 간판, 첫 투자 문구, 푸터에 표시됩니다.
- 회사 이름은 게임 진행상태와 함께 `localStorage`에 자동 저장됩니다.
- 신규 플레이는 현금 `0원`, 법인이 아닌 가족 투자연구소로 시작합니다.
- 이전 버전 저장 데이터가 있으면 투자 진행은 유지하고 회사 이름만 새로 입력받습니다.
- 게임 초기화를 선택하면 회사 이름과 모든 진행 기록을 함께 지웁니다.

### 역사와 데이터

- 시작일은 `2000.01.01`입니다.
- 한 번 진행할 때 하루가 흐릅니다.
- 현재 플레이 기간은 `2000-01-01`부터 `2010-12-31`까지입니다.
- 해당 날짜에 공개되지 않은 미래 사건이나 결과를 미리 노출하면 안 됩니다.
- 주말·휴장일에는 미래 가격을 보간하지 않고 직전 거래일 종가를 유지합니다.
- 실제 수정주가 흐름과 게임용 인수 가격·수익 효과를 혼동시키면 안 됩니다.
- 데이터 출처와 상용화 시 라이선스 계획은 `DATA_SOURCES.md`를 유지·갱신합니다.

## 5. 게임 상태 구조

기존 React 앱의 브라우저 저장 키는 `simul-millennium-capital-v1`이며 상태 스키마 버전은 `3`입니다. Flutter 앱도 같은 논리 키를 사용하며 현재 스키마는 버전 `6`입니다. `shared_preferences`가 플랫폼 저장소에 기록하고 기존 Flutter v1 저장의 회사명·날짜·현금·팀을 v6의 개별 조직 상태로 마이그레이션합니다.

Flutter 앱을 기존 웹 주소에 배포하기 전에는 React 버전 3 브라우저 저장의 포지션·인수·펀드까지 Flutter v6 상태로 변환하는 마이그레이션을 먼저 구현해야 합니다.

```ts
type GameState = {
  version: 3;
  companyName: string;
  currentDate: string;
  cash: number;
  positions: Record<string, { units: number; cost: number }>;
  reputation: number;
  team: number;
  acquisitions: Acquisition[];
  aumHistory: { date: string; value: number }[];
  fundRaised: boolean;
  performanceMonth: string;
  performanceStartAum: number;
  seenEventIds: string[];
};
```

스키마를 변경할 때는 저장 키를 무조건 바꾸지 말고, 기존 버전에서 새 버전으로 변환하는 마이그레이션을 먼저 추가합니다. 사용자의 진행 기록을 조용히 삭제하면 안 됩니다.

## 6. 현재 시장 데이터

### 포함 종목

- 한국: 삼성전자, SK텔레콤, POSCO홀딩스
- 미국: Apple, Microsoft, Cisco
- 일본: Toyota, Sony, SoftBank

### 계산 방식

1. 외부 데이터의 일별 adjusted close를 가져옵니다.
2. 종목별 1999년 12월 마지막 가용 거래일을 100으로 정규화합니다.
3. 포지션은 투자금액을 당시 지수가격으로 나눈 units로 저장합니다.
4. 거래일에는 해당 날짜 값, 주말·휴장일에는 가장 최근 과거 거래일 값을 사용합니다.
5. 미래 날짜 값을 선형 보간하거나 미리 노출하지 않습니다.
6. 다음 날 AUM은 새 지수가격 × 보유 units로 평가합니다.
7. 현재 데이터는 수익률 흐름 검증용이며 당시 실제 화면 호가가 아닙니다.

상용화 전에는 거래소 또는 계약형 공급자의 재배포 권한을 확인해야 합니다. 자세한 내용은 `DATA_SOURCES.md`에 있습니다.

## 7. 로컬 환경에서 시작하는 순서

Node.js `22.13.0` 이상을 사용합니다.

```bash
git clone https://github.com/zzangzzangman2/simul.git
cd simul
npm install
npm run dev
```

PC 브라우저에서 `http://localhost:3000`을 엽니다. PC에서도 세로형 앱 프레임이 가운데 표시되어야 합니다. 정밀 확인 시 개발자 도구의 `390×844` 및 `360×800` 세로 뷰포트를 사용합니다.

## 11. 가족 스토리 현재 구현

새 게임은 `1999.12.31 23:57` 콜드 오픈에서 시작합니다. 주인공 이름과 시작 동기, 가족 투자규칙을 선택하고, 외할아버지에게 현금 대신 첫 장이 0원인 저금장부를 받습니다. 설거지·보호자 동행 문방구 정리·가족 벼룩장터 미니게임으로 첫 조사예산 1만원을 직접 모으며 모든 수입은 장부에 저장됩니다. 가족 신뢰·개별 관계·보호자 권한·회사문화 태그는 저장 스키마 v6에 포함됩니다.

2000~2002년은 엄마·아빠·누나·외할아버지를 조사 조언자로 배치하며 가족 피로도와 도움 기록을 저장합니다. `사람들` 화면은 선택 인물의 상체 초상과 아래 선택 카드를 사용합니다. 정식 채용은 2003년 첫 아르바이트 조사원부터 해금하고 보호자가 계약과 급여를 관리합니다.

첫 안건은 Apple 경영권이 아니라 기업을 어떤 관점으로 조사할지 정하는 `첫 기업 조사노트`입니다. Apple 대체역사 안건은 2007년부터 해금됩니다. 전체 이야기 기준은 `story.md`를 봅니다.

## 12. 대체역사 현재 구현

Flutter에는 Apple 실제 회사명을 사용하는 게임용 체험 시나리오가 구현되어 있습니다. 실제 Apple 시세·내부자료가 아니라 개발용 역사 기준지수와 가상 내부 대화·비용·결과를 사용합니다. 경영권 분기, Project Atlas 제품 제안, 개발 문제, 출시·연기·취소와 지연된 결과가 현금·매출·브랜드·기술·사기·위험·시뮬레이션 기준지수를 바꿉니다.

영구 규칙은 `ALTERNATE_HISTORY.md`, 이벤트 작성은 `CONTENT_GUIDE.md`, 현재 수치는 `BALANCE_NOTES.md`를 봅니다.

## 14. 변경 시 금지사항
