# 부자되기 시뮬레이션 프로젝트 가이드

마지막 갱신: 2026-07-23

## 1. 제품 기준

주 구현은 `flutter_app/`의 Flutter 앱이다. Android와 Web을 대상으로 하며, 웹 `/`는 `/play/index.html`로 이동한다. Next/Vinext는 정적 Flutter 호스트와 서버 전용 뉴스 API를 제공한다.

기준 화면은 모바일 세로 390×844px이다. 최소 360px에서 가로 스크롤이 없어야 하고 PC에서도 최대 430px의 같은 앱 프레임을 가운데 표시한다.

앱은 `처음하기`·`이어하기` 타이틀로 시작한다. 로컬 저장은 최대 5슬롯이며 날짜가 바뀔 때 자동저장하고 허브 메뉴에서 수동저장한다.

## 2. 시작과 생활 루프

- 프롤로그: 1999.12.31 21:40, TV 드라마를 본 주인공의 투자 선언과 가족 대화
- 입문 튜토리얼: 중·고등학생이 함께 듣는 새천년 투자학원에서 회사·매수/매도·시장가/지정가를 학습
- 교육비 채무: 아빠가 먼저 낸 학원비 1,000,000원. 신규 저장에만 생성되며 아빠를 선택해 회사 통장에서 전액 상환
- 첫 플레이: 2000.01.02 일요일 08:00
- 초기자본: 외할아버지 세뱃돈 10,000원(교육용 증권계좌 예수금)
- 창립 인원: 1명
- 첫 주문 권한: 세뱃돈 1만원을 첫 투자금으로 기록한 뒤 가족 승인
- 공간: 작은방, 거실, 부엌
- 일거리: 설거지, 문방구 정리, 가족 벼룩장터

작은방 CRT는 시장, 서류함은 장부, 거실 편지는 안건, 가족 공간은 조직, 부엌 전화는 일거리로 연결한다. 배경에 그려진 물건과 터치 영역이 일치해야 한다.

학원비 채무와 투자금은 다른 계정이다. 상환은 증권 예수금을 자동 출금하지 않고 회사 통장 잔액만 사용하며 `family_debt_repayment` 원장에 기록한다. 기존 저장에는 학원비 채무를 소급 생성하지 않는다.

## 3. 가상시장 절대 규칙

거래 가능한 회사는 모두 게임 전용 가상기업이다.

- 출발 기업 30개의 이름·업종·제품은 고정
- 시장 구분: 미래시장, 도전시장, 신규·분사, 관심
- 새 게임마다 고유 `simulationSeed` 생성
- 시드가 가격, 사건 시기·성패, IPO, 분사, 유증, 상폐를 결정
- 같은 시드와 같은 선택은 같은 결과
- 신규상장과 분사로 종목 수가 30개 이상으로 증가
- 상장폐지와 합병 확장으로 시장 구성은 다시 감소 가능

실제 종가 JSON이나 실종목 목록은 배포하지 않는다. 현실 26년 자료는 `DATA_SOURCES.md`에 출처를 기록하고 업종별 인과 문법만 추출한다.

## 4. 하루 진행과 정보 비대칭

```text
오늘 20:00 종가·공개사건 저장
        ↓
다음 날 08:00 준비 화면
        ├─ Gemini: 전날 공개 사실만 기사화
        └─ 엔진: 오늘 숨은 시나리오를 한 번 생성·저장
        ↓
신문 열림(상태는 이미 다음 날 08:00)
        ↓
선택: 1,200원 조사보고서 구매
        ↓
시간이 revealMinute를 지나면 장중 속보 공개
```

신문은 오늘 사건, 성공 여부, 영향률과 미래 종가를 알 수 없다. 보고서는 최대 3개 현장 징후를 주되 방향과 결과를 숨긴다. Gemini는 가격이나 사건 결과를 결정하지 않는다.

## 5. 시장 시간

- 게임 하루: 08:00~20:00
- 08:00~08:59: 이전 종가 고정, 주문 불가
- 09:00~14:50: 2000~2010년 기준 정규 거래와 1분봉
- 14:50~15:00: 종가 동시호가 구간
- 15:00~20:00: 종가 고정, 추가 분봉 없음
- 시장 화면이 열렸을 때 현실 1초마다 게임 1분 진행
- `1시간 보내기`로 빠른 진행 가능

주말·휴장일에는 직전 종가를 유지한다. 장중 경로는 월드시드·업종·회사·사건 영향을 조합하고 저장된 일별 종가로 닫힌다.

## 6. 저장과 마이그레이션

현재 상태 스키마는 `v15`이다.

- 1번 슬롯은 기존 단일 저장 키와 호환
- 2~5번 슬롯은 파생 키 사용
- 슬롯별 정상 백업, 손상 원문, 최근 저장 시각 보존
- `hiddenMarketScenario`는 날짜별 단회 생성
- 이전 Flutter 저장의 날짜·현금·조직·원장 보존
- 과거 실기업 포지션은 기록 원가를 현금으로 한 번 환급하고 제거
- 과거 제어 회사는 한빛통신으로 전환
- 마이그레이션은 멱등이어야 함

## 8. 이벤트 확장

이벤트는 수천 개 완성 기사를 복사하지 않고 다음 축을 조합한다.

1. 업종 원인 문법
2. 회사 제품·재무 상태
3. 시작·중간·결과·후속 단계
4. 성공·실패·지연·축소 결과
5. 자본·분할·상장·상폐 후속 문법
6. 시장 국면과 월드시드

새 사건은 `CONTENT_GUIDE.md`의 업종 허용표를 통과해야 한다. 현실 레퍼런스는 실제 회사와 일대일 대응시키지 않는다.

## 9. Gemini 서버 경계

인증정보는 서버에만 둔다. Flutter와 브라우저 번들에 키를 넣지 않는다. API 실패·오프라인·타임아웃이면 결정론적 로컬 신문으로 진행한다.

허용 입력: 전날 날짜, 전날 공개 사건, 전날 시장 집계.

금지 입력: 오늘 숨은 시나리오, 보고서 힌트, 영향률, 성공 플래그, 미래 가격, 개인 장부·개인 매매.

## 11. 배포

`npm run build:flutter-web`은 `flutter build web --release --base-href /play/` 산출물을 검증한 뒤 `public/play/`을 교체한다. `npm run build:release`는 Flutter Web과 Vinext를 함께 빌드한다.

실기기 확인은 같은 Wi-Fi에서 개발 서버를 `0.0.0.0:8000`으로 열고 휴대폰에서 `http://<PC 내부 IP>:8000`으로 접속한다.
