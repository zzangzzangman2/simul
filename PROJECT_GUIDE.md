# MILLENNIUM CAPITAL 프로젝트 인수인계 문서

이 문서는 다른 PC, 다른 개발 환경, 다른 에이전트가 저장소를 처음 열어도 현재 구현을 이해하고 이어서 작업할 수 있도록 작성한 기준 문서입니다.

## 1. 제품 한 줄 정의

1999년 마지막 밤의 가족 프롤로그에서 시작해, 10살 소년이 초기자본 100만원과 창립 인원 1명으로 실제 역사적 시장 흐름 속에서 투자연구소를 키우는 모바일 세로형 경영 시뮬레이션입니다.

## 2. 절대 유지해야 하는 제품 요구사항

### 화면과 조작

- 디자인 기준 기기는 모바일 세로 모드 `390×844px`입니다.
- 최소 지원 너비는 `360px`입니다.
- 모바일에서 가로 스크롤이 생기면 실패로 봅니다.
- 주요 버튼은 손가락으로 누르기 쉽도록 최소 44px 높이를 유지합니다.
- 하단의 독립된 `1시간 보내기`·`하루 보내기` 버튼은 모바일 안전영역을 침범하지 않아야 하며 글자가 선명하게 보여야 합니다. 미해결 안건이 있으면 두 버튼을 모두 잠급니다.
- PC 브라우저에서도 전체 플레이와 테스트가 가능해야 합니다.
- PC에서도 폭 430px 이하의 동일한 세로형 앱 프레임을 가운데 표시합니다. 넓은 데스크톱 대시보드로 확장하지 않습니다.
- 가족 프롤로그 완료 후에는 주식 목록이 아니라 2000년 가족 아파트의 작은방에서 시작합니다.
- 아파트 허브는 작은방 ↔ 거실 ↔ 부엌의 세로형 3공간 동선입니다. 작은방 CRT·서류함은 시장·고급 포트폴리오로, 거실 안건 편지·가족 공간은 안건함·가족/조직으로, 부엌 유선전화는 일거리로 연결합니다.
- 메뉴 카드처럼 임의의 위치에 기능을 띄우지 않습니다. 각 핫스폿은 배경에 실제로 그려진 해당 오브젝트의 중심에 맞추고 최소 48px 터치 영역을 유지합니다.
- 부엌 유선전화의 현재 기능은 일거리이며 M&A 전화는 성장 후 별도로 해금합니다.
- 종목은 모바일 1열 목록으로 표시하고, 종목 상세의 `사기`·`팔기` 버튼을 누르면 거래 바텀시트가 열려야 합니다. 주식시장의 화면 뒤로가기와 시스템 Back은 현재 장중 시각 저장이 성공한 뒤 같은 경로로 화면을 닫습니다.
- 닉네임·회사 이름 입력 중 키보드 높이는 한 번만 반영하며, 배경은 고정하고 대화 카드가 부족한 높이에서 스크롤되어야 합니다.

### 첫 진입

- 첫 방문 시 가족 스토리 프롤로그가 가장 먼저 보여야 합니다.
- 방 설명과 엄마·아빠·누나의 대화가 이어진 뒤에 주인공과 투자회사 이름을 정합니다.
- 공백만 입력한 이름으로 시작할 수 없습니다.
- 회사 이름은 최대 24자입니다.
- 설립 후 회사 이름은 접미사를 임의로 더하지 않고 상단 간판, 작은방, 시장 계좌, 사람들, 서류함에 그대로 표시됩니다.
- 회사 이름은 게임 진행상태와 함께 플랫폼 로컬 저장소에 자동 저장됩니다.
- 신규 플레이는 초기자본 `1,000,000원`, 창립 인원 `1명`의 투자회사로 시작합니다. 가족 조언자는 정식 직원 수에 포함하지 않습니다.
- 이전 Flutter 저장은 v9로 마이그레이션하고, React 웹 v3 저장은 지원 필드를 명시적으로 가져옵니다. 이미 이름이 있으면 다시 입력받지 않으며 진행을 조용히 초기화하지 않습니다.
- 게임 초기화를 선택하면 회사 이름과 모든 진행 기록을 함께 지웁니다.

### 역사와 데이터

- 시작일은 `2000.01.01`입니다.
- 하루는 08:00~20:00의 공통 시계로 진행합니다. `1시간 보내기`는 60분을 보내고, `하루 보내기`는 기존 뉴스 생성→신문 확인 흐름을 거친 뒤 다음 날짜 08:00으로 넘어갑니다.
- 현재 플레이 기간은 `2000-01-01`부터 `2010-12-31`까지입니다.
- 해당 날짜에 공개되지 않은 미래 사건이나 결과를 미리 노출하면 안 됩니다.
- 주말·휴장일에는 미래 가격을 보간하지 않고 직전 거래일 종가를 유지합니다.
- 실제 수정주가 흐름과 게임용 인수 가격·수익 효과를 혼동시키면 안 됩니다.
- 데이터 출처와 상용화 시 라이선스 계획은 `DATA_SOURCES.md`를 유지·갱신합니다.

## 3. 현재 구현된 기능

기본 사용자 경로인 Flutter 앱에는 다음이 구현되어 있습니다.

1. 1999년 가족 프롤로그 뒤 주인공·투자규칙·연구소 이름을 정하는 온보딩
2. 초기자본 100만원과 창립 인원 1명, 일거리 미니게임 3종, 가족 조언자 조직
3. 작은방·거실·부엌의 고급 배경 3종과 실제 오브젝트 위치에 맞춘 장소별 핫스폿
4. 08:00~20:00 공통 시계, 행동별 시간비용, 독립된 `1시간 보내기`와 신문 확인을 거치는 `하루 보내기`
5. 국내 기본 팩 22개와 별도 해외 참고 탭 6개, 실제 일별 종가와 휴장일 직전 종가 유지. 해외 주문은 실제 환율 원장을 연결하기 전까지 잠금
6. 실제 종가로 끝나는 게임용 장중 틱과 국내·KOSPI·KOSDAQ·해외 필터
7. 실제 일별 종가에 수렴하는 현재 장중 게임 체결가 기반 매수·매도, 거래비용, 현금·포지션·AUM 저장과 가죽 장부형 고급 포트폴리오
8. Flutter v9 로컬 자동 저장, Flutter v8 보존 마이그레이션, 주식시장 시스템 Back 저장과 기존 React 웹 v3 진행 가져오기
9. 날짜별 역사·생활 소식, 실제 경영진 카드와 일간 신문
10. 2007년부터 열리는 Apple 게임용 대체역사 세로 슬라이스
11. PC에서도 동일하게 보이는 최대 430px 세로형 앱 프레임

평판, 펀드, 인수 딜과 장기 캠페인 로직은 기존 React 기준 구현에 남아 있으며 Flutter 이관이 완료된 기능처럼 설명하지 않습니다.

### Flutter 전환 상태

2026-07-21부터 새 주 구현은 `flutter_app/`에서 진행합니다. 기존 React 앱은 시장·거래·인수 로직과 저장 데이터의 회귀 비교용 기준 구현이며, 새 사용자 기능을 이중 구현하지 않습니다. 삭제 여부는 v9 가져오기와 회귀 검증을 대체할 절차를 마련한 뒤 별도로 결정합니다.

## 5. 게임 상태 구조

기존 React 앱의 브라우저 저장 키는 `simul-millennium-capital-v1`이며 상태 스키마 버전은 `3`입니다. Flutter 앱도 같은 논리 키를 사용하며 현재 스키마는 버전 `9`입니다. `shared_preferences`가 플랫폼 저장소에 기록하고 이전 Flutter 저장을 v9로 마이그레이션하며, 웹에서는 React v3의 회사명·날짜·현금·포지션과 숫자형 팀을 명시적으로 변환해 가져옵니다. 아직 Flutter 대응 모델이 없는 React 전용 필드는 가져온 것으로 표시하지 않습니다.

현재 웹 기본 주소는 Flutter 게임으로 시작합니다. 가져올 React v3 저장이 있으면 v9 구조로 한 번 변환하고, 이미 Flutter 저장이 있으면 그 진행을 우선합니다. 이전 저장을 조용히 삭제하거나 같은 원장을 중복 반영하면 안 됩니다.

초기자본 변경 전 생성된 Flutter v8의 0원 저장은 포지션·원장·일거리 수입 흔적이 없을 때만 100만원을 한 번 지급합니다. 날짜·시각·안건·가족 도움 같은 비금융 진행은 그대로 보존하며, 금융 활동 흔적이 있거나 v9로 저장된 상태에는 자본을 다시 지급하지 않습니다. 모든 변경은 플랫폼 저장 성공 후 화면에 게시하고, 저장 실패 시 기존 화면 상태와 기존 영속 데이터를 유지하며 사용자에게 실패를 알립니다.

```ts
type GameState = {
  version: 3;
  companyName: string;
  currentDate: string;
  cash: number;
  positions: Record<string, { units: number; cost: number }>;
  reputation: number;
  team: number;
  acquisitions: Acquisition[];
  aumHistory: { date: string; value: number }[];
  fundRaised: boolean;
  performanceMonth: string;
  performanceStartAum: number;
  seenEventIds: string[];
};
```

스키마를 변경할 때는 저장 키를 무조건 바꾸지 말고, 기존 버전에서 새 버전으로 변환하는 마이그레이션을 먼저 추가합니다. 사용자의 진행 기록을 조용히 삭제하면 안 됩니다.

## 6. 현재 시장 데이터

### 현재 기본 팩

- 국내: KOSPI 19개, KOSDAQ 3개
- 해외 보존: NASDAQ 3개, TSE 3개
- 해외 거래: USD·JPY를 원화에 그대로 더하지 않도록 실제 환율 원장이 마련될 때까지 참고 전용
- 기본 주가창: 국내
- 필터: 국내, 코스피, 코스닥, 해외
- 가격: `prices`의 실제 일별 종가
- 보조 성과지수: `adjustedIndex`의 기준일=100 수정주가

거래일에는 장중 틱을 자동 생성하며 15:30의 150번째 값과 20:00의 240번째 값은 실제 종가와 정확히 같습니다. 종목의 첫 관측일 전에는 가격을 노출하지 않고 주말·휴장일에는 직전 종가를 유지합니다.

### 전 종목 확장

KRX 전 종목은 로그인형 `scripts/fetch-krx-full-market.py`로 생성합니다. 모든 평일 상장목록 합집합으로 상장폐지와 합병 종목을 보존하며 실패가 있으면 결과 파일을 갱신하지 않습니다. KRX 계정·라이선스 없이 기본 팩을 전 종목이라고 부르면 안 됩니다.

## 7. 로컬 환경에서 시작하는 순서

## 11. 가족 스토리 현재 구현

새 게임은 `1999.12.31 23:57` 콜드 오픈에서 시작합니다. 주인공 이름과 시작 동기, 가족 투자규칙을 선택하고 `2000.01.01`에 초기자본 100만원·창립 인원 1명으로 투자연구소를 엽니다. 설거지·보호자 동행 문방구 정리·가족 벼룩장터 미니게임 수입도 장부에 저장됩니다. 가족 신뢰·개별 관계·보호자 권한·회사문화 태그와 투자 원장은 저장 스키마 v9에 포함됩니다.

## 일중 진행 루프

플레이 하루는 08:00에 시작한다. `1시간 보내기`로 60분씩 최대 20:00까지 진행하며 15:20 장마감 동시호가, 15:30 종가 확정, 15:40~20:00 NXT형 확장장을 거친다. `하루 보내기`는 `한국경제신문` 게임 특별판 생성·확인을 거친 뒤 다음 날 08:00으로 이동한다. 미해결 안건이 있으면 두 진행 버튼이 모두 잠기며, 시장이 휴장해도 하루 생활 콘텐츠와 신문은 유지한다.

## 장소 기반 화면 연출과 시간 비용

중요 기능은 세계 안의 장소로 연결한다. 작은방의 CRT와 서류함에서 주식시장·종목 상세와 고급 포트폴리오·현금원장, 거실의 안건 편지와 가족 공간에서 안건함·가족회의와 가족/조직, 부엌의 유선전화에서 일거리가 열린다. 배경에 없는 물건 위에 기능을 두지 않으며 모든 핫스폿은 실제 오브젝트의 위치와 맞춘다. 장면 상단에는 장소명·현재 시각 안내가 함께 표시된다.

단순 열람과 화면 이동은 무료다. 안건 결정을 확정하면 30분, 가족 도움을 확정하면 30분, 미니게임 보상을 확정하면 60분이 흐른다. 시장 관찰은 생성 틱과 연동되고 빠른 진행은 1시간이다. 모든 장소는 저장 상태의 동일한 시계를 사용한다.

주식시장에서 생성 틱으로 흐른 시각은 화면 뒤로가기와 시스템 Back 모두 같은 저장 가드로 확정한다. 저장 성공 전에는 route를 닫지 않고, 저장 중 반복 Back은 무시하며, 실패하면 시장 화면을 유지해 시각과 진행이 갈라지지 않게 한다.

## 동적 뉴스 생성 구조

하루 종료 시 거실 편집 마감 장면에서 `뉴스를 생성 중입니다`를 표시한다. Flutter는 현재 연도, 플레이어의 최근 행동, 관련 회사, 당일 역사·생활 흐름을 `/api/news`에 보낸다. 서버는 Gemini 3.5 Flash에 경제 기자 역할과 현재 연도 제한을 부여하고 `headline`, `content`, `marketSentiment`, `stockImpactScore`만 JSON Schema로 받는다.

AI는 기사 서사만 담당한다. 실제 종가, 틱의 종가 도달, 주문, 현금과 AUM 계산은 기존 결정론적 코드가 담당한다. AI의 영향 점수는 -30~+50 범위의 향후 게임 가격 엔진 입력값이며 현재는 기사에 표시만 하고 실제 종가를 변경하지 않는다. 서버 장애·오프라인·타임아웃이면 기존 실제 종가 기반 한국경제신문 특별판으로 자동 전환한다.
