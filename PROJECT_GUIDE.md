# MILLENNIUM CAPITAL 프로젝트 인수인계 문서

이 문서는 다른 PC, 다른 개발 환경, 다른 에이전트가 저장소를 처음 열어도 현재 구현을 이해하고 이어서 작업할 수 있도록 작성한 기준 문서입니다.

## 1. 제품 한 줄 정의

1999년 마지막 밤의 가족 프롤로그에서 시작해, 10살 소년이 0원짜리 저금장부와 가족의 신뢰로 종잣돈을 만들고 실제 역사적 시장 흐름 속에서 투자연구소를 키우는 모바일 세로형 경영 시뮬레이션입니다.

## 2. 절대 유지해야 하는 제품 요구사항

### 화면과 조작

- 디자인 기준 기기는 모바일 세로 모드 `390×844px`입니다.
- 최소 지원 너비는 `360px`입니다.
- 모바일에서 가로 스크롤이 생기면 실패로 봅니다.
- 주요 버튼은 손가락으로 누르기 쉽도록 최소 44px 높이를 유지합니다.
- 하단의 `1시간 뒤`·`오늘 신문` 버튼은 모바일 안전영역을 침범하지 않아야 하며 밝은 버튼 위 글자는 검정색으로 선명하게 보여야 합니다.
- PC 브라우저에서도 전체 플레이와 테스트가 가능해야 합니다.
- PC에서도 폭 430px 이하의 동일한 세로형 앱 프레임을 가운데 표시합니다. 넓은 데스크톱 대시보드로 확장하지 않습니다.
- 가족 프롤로그 완료 후에는 주식 목록이 아니라 2000년 가족 아파트의 작은방에서 시작합니다.
- 주식시장은 방 안의 CRT 컴퓨터를 눌러야 열립니다.
- 유선전화는 초반 가족·증권사 연락, 서류함은 포트폴리오로 연결되는 방 기반 허브를 유지합니다. 투자 제안과 M&A 전화는 성장 뒤 해금합니다.
- 종목은 모바일 1열 목록으로 표시하고, 종목 상세의 `사기`·`팔기` 버튼을 누르면 거래 바텀시트가 열려야 합니다.

### 첫 진입

- 첫 방문 시 가족 스토리 프롤로그가 가장 먼저 보여야 합니다.
- 방 설명과 엄마·아빠·누나의 대화가 이어진 뒤에 주인공과 가족 투자연구소 이름을 정합니다.
- 공백만 입력한 이름으로 시작할 수 없습니다.
- 회사 이름은 최대 24자입니다.
- 설립 후 회사 이름은 상단 간판, 첫 투자 문구, 푸터에 표시됩니다.
- 회사 이름은 게임 진행상태와 함께 플랫폼 로컬 저장소에 자동 저장됩니다.
- 신규 플레이는 현금 `0원`, 법인이 아닌 가족 투자연구소로 시작합니다.
- 이전 Flutter 저장은 v8로 마이그레이션하고, React 웹 v3 저장은 지원 필드를 명시적으로 가져옵니다. 이미 이름이 있으면 다시 입력받지 않으며 진행을 조용히 초기화하지 않습니다.
- 게임 초기화를 선택하면 회사 이름과 모든 진행 기록을 함께 지웁니다.

### 역사와 데이터

- 시작일은 `2000.01.01`입니다.
- 하루는 08:00~20:00의 공통 시계로 진행하며, `1시간 뒤`와 결과 행동으로 시간이 흐릅니다. `오늘 신문`을 확인해야 다음 날짜 08:00으로 넘어갑니다.
- 현재 플레이 기간은 `2000-01-01`부터 `2010-12-31`까지입니다.
- 해당 날짜에 공개되지 않은 미래 사건이나 결과를 미리 노출하면 안 됩니다.
- 주말·휴장일에는 미래 가격을 보간하지 않고 직전 거래일 종가를 유지합니다.
- 실제 수정주가 흐름과 게임용 인수 가격·수익 효과를 혼동시키면 안 됩니다.
- 데이터 출처와 상용화 시 라이선스 계획은 `DATA_SOURCES.md`를 유지·갱신합니다.

## 3. 현재 구현된 기능

기본 사용자 경로인 Flutter 앱에는 다음이 구현되어 있습니다.

1. 1999년 가족 프롤로그 뒤 주인공·투자규칙·연구소 이름을 정하는 온보딩
2. 현금 0원과 빈 저금장부, 일거리 미니게임 3종, 가족 조언자 조직
3. 가족 아파트 작은방 허브와 CRT 컴퓨터·유선전화·서류함
4. 08:00~20:00 공통 시계, 행동별 시간비용, `1시간 뒤`와 일간 신문을 거치는 날짜 진행
5. 국내 기본 팩 22개와 별도 해외 참고 탭 6개, 실제 일별 종가와 휴장일 직전 종가 유지. 해외 주문은 실제 환율 원장을 연결하기 전까지 잠금
6. 실제 종가로 끝나는 게임용 장중 틱과 국내·KOSPI·KOSDAQ·해외 필터
7. 실제 일별 종가에 수렴하는 현재 장중 게임 체결가 기반 매수·매도, 거래비용, 현금·포지션·AUM 저장과 포트폴리오
8. Flutter v8 로컬 자동 저장과 기존 React 웹 v3 진행 가져오기
9. 날짜별 역사·생활 소식, 실제 경영진 카드와 일간 신문
10. 2007년부터 열리는 Apple 게임용 대체역사 세로 슬라이스
11. PC에서도 동일하게 보이는 최대 430px 세로형 앱 프레임

평판, 펀드, 인수 딜과 장기 캠페인 로직은 기존 React 기준 구현에 남아 있으며 Flutter 이관이 완료된 기능처럼 설명하지 않습니다.

### Flutter 전환 상태

2026-07-21부터 새 주 구현은 `flutter_app/`에서 진행합니다. 기존 React 앱은 시장·거래·인수 로직과 저장 데이터의 회귀 비교용 기준 구현이며, 새 사용자 기능을 이중 구현하지 않습니다. 삭제 여부는 v8 가져오기와 회귀 검증을 대체할 절차를 마련한 뒤 별도로 결정합니다.

## 5. 게임 상태 구조

기존 React 앱의 브라우저 저장 키는 `simul-millennium-capital-v1`이며 상태 스키마 버전은 `3`입니다. Flutter 앱도 같은 논리 키를 사용하며 현재 스키마는 버전 `8`입니다. `shared_preferences`가 플랫폼 저장소에 기록하고 이전 Flutter 저장을 v8로 마이그레이션하며, 웹에서는 React v3의 회사명·날짜·현금·포지션과 숫자형 팀을 명시적으로 변환해 가져옵니다. 아직 Flutter 대응 모델이 없는 React 전용 필드는 가져온 것으로 표시하지 않습니다.

현재 웹 기본 주소는 Flutter 게임으로 시작합니다. 가져올 React v3 저장이 있으면 v8 구조로 한 번 변환하고, 이미 Flutter 저장이 있으면 그 진행을 우선합니다. 이전 저장을 조용히 삭제하거나 같은 원장을 중복 반영하면 안 됩니다.

```ts
type GameState = {
  version: 3;
  companyName: string;
  currentDate: string;
  cash: number;
  positions: Record<string, { units: number; cost: number }>;
  reputation: number;
  team: number;
  acquisitions: Acquisition[];
  aumHistory: { date: string; value: number }[];
  fundRaised: boolean;
  performanceMonth: string;
  performanceStartAum: number;
  seenEventIds: string[];
};
```

스키마를 변경할 때는 저장 키를 무조건 바꾸지 말고, 기존 버전에서 새 버전으로 변환하는 마이그레이션을 먼저 추가합니다. 사용자의 진행 기록을 조용히 삭제하면 안 됩니다.

## 6. 현재 시장 데이터

### 현재 기본 팩

- 국내: KOSPI 19개, KOSDAQ 3개
- 해외 보존: NASDAQ 3개, TSE 3개
- 해외 거래: USD·JPY를 원화에 그대로 더하지 않도록 실제 환율 원장이 마련될 때까지 참고 전용
- 기본 주가창: 국내
- 필터: 국내, 코스피, 코스닥, 해외
- 가격: `prices`의 실제 일별 종가
- 보조 성과지수: `adjustedIndex`의 기준일=100 수정주가

거래일에는 장중 틱을 자동 생성하며 15:30의 150번째 값과 20:00의 240번째 값은 실제 종가와 정확히 같습니다. 종목의 첫 관측일 전에는 가격을 노출하지 않고 주말·휴장일에는 직전 종가를 유지합니다.

### 전 종목 확장

KRX 전 종목은 로그인형 `scripts/fetch-krx-full-market.py`로 생성합니다. 모든 평일 상장목록 합집합으로 상장폐지와 합병 종목을 보존하며 실패가 있으면 결과 파일을 갱신하지 않습니다. KRX 계정·라이선스 없이 기본 팩을 전 종목이라고 부르면 안 됩니다.

## 7. 로컬 환경에서 시작하는 순서

Node.js `22.13.0` 이상을 사용합니다.

```bash
git clone https://github.com/zzangzzangman2/simul.git
cd simul
npm install
npm run dev
```

`npm run dev`는 Vertex ADC를 사용할 수 있는 Next Node 개발 서버를 실행합니다. Sites/Vinext 개발 환경만 별도로 확인할 때는 `npm run dev:sites`를 사용합니다.

Flutter 소스를 바꾼 뒤 배포용 `/play/` 번들을 갱신하려면 `npm run build:flutter-web`을 사용합니다. 이 스크립트는 `--base-href /play/` 빌드와 필수 산출물·저장 브리지 검증을 마친 뒤에만 `public/play/`을 교체합니다.

PC 브라우저에서 `http://localhost:3000`을 엽니다. PC에서도 세로형 앱 프레임이 가운데 표시되어야 합니다. 정밀 확인 시 개발자 도구의 `390×844` 및 `360×800` 세로 뷰포트를 사용합니다.

## 8. 데이터 갱신 순서

```bash

```

갱신 후 반드시 확인합니다.

3. 원시 종가와 보조 수정주가 지수 차이가 분할·배당·합병 때문인지 확인
4. `generatedAt`, 기준일, 원본 공급자 정보를 확인
5. 빌드와 테스트 후 변경 데이터를 커밋

## 11. 가족 스토리 현재 구현

새 게임은 `1999.12.31 23:57` 콜드 오픈에서 시작합니다. 주인공 이름과 시작 동기, 가족 투자규칙을 선택하고, 외할아버지에게 현금 대신 첫 장이 0원인 저금장부를 받습니다. 설거지·보호자 동행 문방구 정리·가족 벼룩장터 미니게임으로 첫 조사예산 1만원을 직접 모으며 모든 수입은 장부에 저장됩니다. 가족 신뢰·개별 관계·보호자 권한·회사문화 태그와 투자 원장은 저장 스키마 v8에 포함됩니다.

2000~2002년은 엄마·아빠·누나·외할아버지를 조사 조언자로 배치하며 가족 피로도와 도움 기록을 저장합니다. `사람들` 화면은 선택 인물의 상체 초상과 아래 선택 카드를 사용합니다. 정식 채용은 2003년 첫 아르바이트 조사원부터 해금하고 보호자가 계약과 급여를 관리합니다.

첫 안건은 Apple 경영권이 아니라 기업을 어떤 관점으로 조사할지 정하는 `첫 기업 조사노트`입니다. Apple 대체역사 안건은 2007년부터 해금됩니다. 전체 이야기 기준은 `story.md`를 봅니다.

## 12. 대체역사 현재 구현

Flutter에는 Apple 실제 회사명을 사용하는 게임용 체험 시나리오가 구현되어 있습니다. 실제 Apple 시세·내부자료가 아니라 개발용 역사 기준지수와 가상 내부 대화·비용·결과를 사용합니다. 경영권 분기, Project Atlas 제품 제안, 개발 문제, 출시·연기·취소와 지연된 결과가 현금·매출·브랜드·기술·사기·위험·시뮬레이션 기준지수를 바꿉니다.

## 장소 기반 화면 연출과 시간 비용

중요 기능은 세계 안의 장소로 연결한다. 주식시장과 종목 상세는 소년 방 CRT, 안건은 거실 가족회의, 하루 결산 신문은 저녁 거실 식탁, 일거리는 집과 동네에서 열린다. 장면 상단에는 장소명·현재 시각·행동 시간 안내가 함께 표시된다.

단순 열람과 화면 이동은 무료다. 안건 결정을 확정하면 30분, 가족 도움을 확정하면 30분, 미니게임 보상을 확정하면 60분이 흐른다. 시장 관찰은 생성 틱과 연동되고 빠른 진행은 1시간이다. 모든 장소는 저장 상태의 동일한 시계를 사용한다.

## 동적 뉴스 생성 구조

하루 종료 시 거실 편집 마감 장면에서 `뉴스를 생성 중입니다`를 표시한다. Flutter는 현재 연도, 플레이어의 최근 행동, 관련 회사, 당일 역사·생활 흐름을 `/api/news`에 보낸다. 서버는 Gemini 3.5 Flash에 경제 기자 역할과 현재 연도 제한을 부여하고 `headline`, `content`, `marketSentiment`, `stockImpactScore`만 JSON Schema로 받는다.

AI는 기사 서사만 담당한다. 실제 종가, 틱의 종가 도달, 주문, 현금과 AUM 계산은 기존 결정론적 코드가 담당한다. AI의 영향 점수는 -30~+50 범위의 향후 게임 가격 엔진 입력값이며 현재는 기사에 표시만 하고 실제 종가를 변경하지 않는다. 서버 장애·오프라인·타임아웃이면 기존 실제 종가 기반 한국경제신문 특별판으로 자동 전환한다.
