# MILLENNIUM CAPITAL 프로젝트 인수인계 문서

이 문서는 다른 PC, 다른 개발 환경, 다른 에이전트가 저장소를 처음 열어도 현재 구현을 이해하고 이어서 작업할 수 있도록 작성한 기준 문서입니다.

## 1. 제품 한 줄 정의

2000년 1월 1일부터 실제 역사적 시장 흐름을 따라 상장회사에 투자하고, 팀과 평판을 키우며, 실제 M&A에서 착안한 인수 기회를 실행하는 모바일 세로형 투자회사 경영 시뮬레이션입니다.

## 2. 절대 유지해야 하는 제품 요구사항

### 화면과 조작

- 디자인 기준 기기는 모바일 세로 모드 `390×844px`입니다.
- 최소 지원 너비는 `360px`입니다.
- 모바일에서 가로 스크롤이 생기면 실패로 봅니다.
- 주요 버튼은 손가락으로 누르기 쉽도록 최소 44px 높이를 유지합니다.
- 하단의 `한 달 진행` 버튼은 모바일 안전영역을 침범하지 않아야 합니다.
- PC 브라우저에서도 전체 플레이와 테스트가 가능해야 합니다.
- PC에서도 폭 430px 이하의 동일한 세로형 앱 프레임을 가운데 표시합니다. 넓은 데스크톱 대시보드로 확장하지 않습니다.
- 메인 게임은 상단 회사명·날짜, 자산 요약, 현재 탭 콘텐츠, 월 진행 버튼, 하단 내비게이션 순서입니다.
- 종목은 모바일 1열 목록으로 표시하고, 종목을 누르면 거래 바텀시트가 즉시 열려야 합니다.
- 시장·포트폴리오·딜룸 메뉴는 하단에 고정하며 모바일 안전영역을 반영합니다.

### 첫 진입

- 첫 방문 시 회사 이름 입력 화면이 가장 먼저 보여야 합니다.
- 공백만 입력한 이름으로 시작할 수 없습니다.
- 회사 이름은 최대 24자입니다.
- 설립 후 회사 이름은 상단 간판, 첫 투자 문구, 푸터에 표시됩니다.
- 회사 이름은 게임 진행상태와 함께 `localStorage`에 자동 저장됩니다.
- 이전 버전 저장 데이터가 있으면 투자 진행은 유지하고 회사 이름만 새로 입력받습니다.
- 게임 초기화를 선택하면 회사 이름과 모든 진행 기록을 함께 지웁니다.

### 역사와 데이터

- 시작일은 `2000.01.01`입니다.
- 한 번 진행할 때 한 달이 흐릅니다.
- 현재 플레이 기간은 `2000-01`부터 `2010-12`까지입니다.
- 해당 월에 공개되지 않은 미래 사건이나 결과를 미리 노출하면 안 됩니다.
- 실제 수정주가 흐름과 게임용 인수 가격·수익 효과를 혼동시키면 안 됩니다.
- 데이터 출처와 상용화 시 라이선스 계획은 `DATA_SOURCES.md`를 유지·갱신합니다.

## 5. 게임 상태 구조

브라우저 저장 키는 `simul-millennium-capital-v1`입니다. 저장 객체의 현재 스키마 버전은 `2`입니다.

```ts
type GameState = {
  version: 2;
  companyName: string;
  monthIndex: number;
  cash: number;
  positions: Record<string, { units: number; cost: number }>;
  reputation: number;
  team: number;
  acquisitions: Acquisition[];
  aumHistory: { month: string; value: number }[];
  fundRaised: boolean;
};
```

스키마를 변경할 때는 저장 키를 무조건 바꾸지 말고, 기존 버전에서 새 버전으로 변환하는 마이그레이션을 먼저 추가합니다. 사용자의 진행 기록을 조용히 삭제하면 안 됩니다.

## 6. 현재 시장 데이터

### 포함 종목

- 한국: 삼성전자, SK텔레콤, POSCO홀딩스
- 미국: Apple, Microsoft, Cisco
- 일본: Toyota, Sony, SoftBank

### 계산 방식

1. 외부 데이터의 월별 adjusted close를 가져옵니다.
2. 종목별 첫 가용 기준월을 100으로 정규화합니다.
3. 포지션은 투자금액을 당시 지수가격으로 나눈 units로 저장합니다.
4. 다음 달 AUM은 새 지수가격 × 보유 units로 평가합니다.
5. 현재 데이터는 수익률 흐름 검증용이며 당시 실제 화면 호가가 아닙니다.

상용화 전에는 거래소 또는 계약형 공급자의 재배포 권한을 확인해야 합니다. 자세한 내용은 `DATA_SOURCES.md`에 있습니다.

## 7. 로컬 환경에서 시작하는 순서

Node.js `22.13.0` 이상을 사용합니다.

```bash
git clone https://github.com/zzangzzangman2/simul.git
cd simul
npm install
npm run dev
```

PC 브라우저에서 `http://localhost:3000`을 엽니다. PC에서도 세로형 앱 프레임이 가운데 표시되어야 합니다. 정밀 확인 시 개발자 도구의 `390×844` 및 `360×800` 세로 뷰포트를 사용합니다.

## 8. 데이터 갱신 순서

```bash
npm run data:fetch
```

갱신 후 반드시 확인합니다.

1. 9개 종목 모두 `2000-01`과 `2010-12` 값을 갖는지 확인
2. 각 종목이 게임 기간 132개월을 갖는지 확인
3. 이전 스냅샷 대비 급격한 수정주가 변화가 기업행동 때문인지 확인
4. `generatedAt`, 기준월, 원본 공급자 정보를 확인
5. 빌드와 테스트 후 변경 데이터를 커밋

## 9. 필수 검증 절차

### 자동 검증

```bash
npm run build
npm test
```

### 모바일 세로 검증

1. 새 브라우저 저장 상태에서 회사명 입력 화면이 보이는지 확인
2. 빈 이름일 때 `회사 설립하기`가 비활성화되는지 확인
3. 이름 입력 후 게임 화면으로 진입하는지 확인
4. 새로고침 후 회사 이름과 진행상태가 유지되는지 확인
5. `390×844` 및 `360×800`에서 가로 스크롤이 없는지 확인
6. 하단 월 진행 버튼이 화면 안전영역 위에 보이는지 확인
7. 회사 이름이 길어도 상단 레이아웃이 깨지지 않는지 확인
8. 첫 화면에서 큰 소개 문구 없이 자산 요약과 종목 목록이 바로 보이는지 확인
9. 종목 목록이 1열이고 종목을 누르면 거래 바텀시트가 아래에서 열리는지 확인
10. 시장·포트폴리오·딜룸 하단 내비게이션이 스크롤 중에도 유지되는지 확인
11. PC 넓은 화면에서도 앱 프레임이 430px보다 넓어지지 않는지 확인

### 플레이 검증

1. 종목 선택
2. 25% 매수
3. 현금 감소와 포지션 생성 확인
4. 한 달 진행
5. 날짜와 AUM 변경 확인
6. 포트폴리오 탭에서 수익률 확인
7. 조건 충족 전 딜이 잠겨 있는지 확인
8. 게임 초기화 후 다시 회사명 입력 화면으로 돌아가는지 확인

## 12. 변경 시 금지사항

- 모바일 화면을 확인하지 않고 데스크톱만 맞추지 않습니다.
- 실제 데이터처럼 보이는 임의 가격을 출처 없이 추가하지 않습니다.
- 미래 사건을 이전 월에 노출하지 않습니다.
- 사용자의 기존 저장 데이터를 마이그레이션 없이 폐기하지 않습니다.
- 출처와 게임화 안내 문구를 삭제하지 않습니다.
- 빌드 또는 테스트가 깨진 상태로 푸시하지 않습니다.
