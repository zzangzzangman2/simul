# 피해야 할 것

이 문서는 구현 중 반복해서 생길 수 있는 잘못된 방향을 모아 둡니다. 짧고 절대적인 규칙은 `AGENTS.md`가 최우선이며, 충돌할 경우 `AGENTS.md`를 따릅니다.

## 제품과 화면

- 모바일 확인 없이 데스크톱 화면만 맞추지 않습니다.
- PC에서 넓은 대시보드로 확장하지 않습니다.
- 360px 화면에서 가로 스크롤이 생기게 하지 않습니다.
- 주요 행동을 작은 글자 링크나 44px 미만의 터치 영역에 숨기지 않습니다.
- 첫 화면에 구 React 투자회사 설립 온보딩을 다시 노출하지 않습니다.
- 가족 프롤로그 완료 직후 주식 목록을 먼저 보여주지 않습니다.
- 방 허브를 의미 없는 장식으로 만들거나 모든 기능을 일반 메뉴로 되돌리지 않습니다.
- 하단 `1시간 뒤`·`오늘 신문` 버튼의 글자 대비와 모바일 안전영역을 희생하지 않습니다.

## 주인공과 이야기

- 10살 소년을 성인 체형, 성인 의상 또는 지나치게 어두운 범죄자 이미지로 표현하지 않습니다.
- 실제 시스템 침입 절차, 도구 사용법이나 따라 할 수 있는 해킹 지침을 게임에 넣지 않습니다.
- 불법 행위를 아무 결과 없이 영웅적으로만 포장하지 않습니다.
- 시대에 맞지 않는 스마트폰, 무선 이어버드와 현대 UI를 2000년 장면에 넣지 않습니다.
- 미래 사건이나 성공 기업의 결과를 해당 날짜보다 먼저 암시하지 않습니다.

## 종잣돈과 미니게임

- 새 게임에서 시작 자본이나 큰 투자금을 선물로 즉시 지급하지 않습니다.
- 10살 주인공을 일반 사업장에 무허가로 고용하거나 야간·위험 노동에 투입하지 않습니다.
- 시대 기준 임금과 게임용 집안일 용돈·중고 판매 수입을 같은 공식 통계처럼 표시하지 않습니다.
- 일거리 버튼을 한 번 누른 뒤 결과문만 띄우고 돈을 지급하지 않습니다. 실제 조작 규칙, 실수와 숙련 점수가 있어야 합니다.
- 일거리 수입을 무한 반복할 수 있게 두지 않으며 하루 제한과 공부·가족 시간의 기회비용을 유지합니다.
- 첫 목표 달성 뒤 일거리를 강제로 반복시키거나 메뉴를 없애지 않습니다.
## 가족과 직원

- 2000~2002년에 성인 정식 직원을 고용하거나 10살 주인공이 직접 근로계약·급여 지급을 하지 않습니다.
- 가족 도움을 무료 노동이나 무한 반복 보너스로 취급하지 않습니다.
- 직원 등급을 인간의 가치·인격·윤리 등급으로 표현하거나 `F급 인간`처럼 모욕적으로 부르지 않습니다.
- 등급 색상만으로 상태를 구분하지 않고 문자와 설명을 함께 표시합니다.
- 능력이 높다는 이유로 비윤리적 행동을 정당화하거나 사건을 자동으로 덮지 않습니다.
- 직원을 즉시 해고하고 끝나는 소모품으로 만들지 않으며 성장·재배치·교육·퇴사 여파를 남깁니다.
- 직원 배치의 대표 인물 이미지를 전신 위주로 작게 보여 주지 않습니다. 선택 인물은 상체 초상을 크게, 후보는 아래 카드로 보여 줍니다.
## 데이터

- 출처 없는 임의 가격을 실제 데이터처럼 표시하지 않습니다.
- 주말·휴장일 가격을 미래 거래일 값으로 보간하지 않습니다.
- 수정주가 지수를 당시 실제 호가라고 설명하지 않습니다.
- 게임이 자동 생성한 장중 틱을 실제 체결가·시가·고가·저가라고 설명하지 않습니다.
- KRX 전 종목 수집이 끝나기 전 국내 기본 팩 22개를 당시 상장사 전체라고 부르지 않습니다.
- 실제 가격과 게임용 인수 가격·효과를 같은 종류의 수치처럼 섞지 않습니다.
- 데이터 출처, 수집일과 라이선스 주의사항을 삭제하지 않습니다.
- 생존한 기업만 남겨 상장폐지와 합병 이력을 지우지 않습니다.

## 기술과 성능

- 버벅임의 원인을 측정하기 전에 Unity로 전면 이전하지 않습니다.
- 원화 크기의 고해상도 PNG를 모든 애니메이션 프레임에 그대로 사용하지 않습니다.
- 작은 캐릭터 파일을 각각 따로 그려 드로우콜을 불필요하게 늘리지 않습니다.
- 과도한 실시간 블러, 파티클, 조명과 반복 렌더링으로 모바일 성능을 희생하지 않습니다.
- 상태 스키마를 바꾸면서 저장 키만 새로 만들어 기존 세이브를 버리지 않습니다.
- 시장 데이터 전체를 매 렌더마다 다시 계산하거나 파싱하지 않습니다.

## 작업 방식과 문서

- 사용자 변경사항을 확인하지 않고 파일을 덮어쓰지 않습니다.
- 완료하지 않은 기능을 `WORK_LOG.md`에 완료로 기록하지 않습니다.
- 같은 규칙을 여러 문서에 복사해 서로 다르게 유지하지 않습니다.
- 중요한 방향 변경은 이유 없이 코드에만 남기지 말고 `DECISIONS.md`에 기록합니다.
- 작업 종료 전에 `HANDOFF.md`의 현재 상태와 다음 한 가지 작업을 갱신하지 않은 채 넘기지 않습니다.
- 빌드나 테스트가 실패한 상태를 숨기거나 `main`에 푸시하지 않습니다.

## 시장 시간·신문 금지사항

- `15:20 장 마감`이라고 쓰지 않는다. 15:20은 장마감 동시호가 시작, 15:30이 정규장 마감이다.
- 현재 구현에 NXT형 확장장을 다시 넣지 않는다. 15:30 실제 종가 확정 뒤에는 20:00까지 추가 주문·가격 변동·분봉 없이 종가를 유지한다.
- 하루 넘김과 신문 노출 순서를 뒤집지 않는다. 먼저 현재 날짜 신문을 보여주고 확인 후 다음 날 08:00으로 이동한다.
- 신문에 다음 거래일 가격, 미래 사건, 아직 공개되지 않은 CEO/회사 정보를 미리 싣지 않는다.
- 15:30 실제 종가 앵커를 장중 임의 틱으로 덮어쓰지 않는다.

## 장소와 시간 연출 금지사항

- 주식시장, 하루 결산 신문, 중요 이벤트를 맥락 없는 흰색 일반 팝업으로 되돌리지 않는다.
- 단순히 메뉴를 열거나 뒤로 가는 행동에 시간을 차감하지 않는다.
- 미니게임을 시작만 하고 취소했는데 60분을 차감하지 않는다. 완료 보상을 확정한 뒤에만 시간을 진행한다.
- 각 화면이 서로 다른 시계를 가지게 하지 않는다. 저장 상태의 한 시각을 모든 장소가 공유한다.
- 360px에서 장소명·현재 시간을 숨기지 않는다. 공간이 부족하면 부가 설명이나 비용 배지를 먼저 줄인다.

## 동적 뉴스 AI 금지사항

- API 키, 서비스 계정 JSON, 액세스 토큰을 Flutter·브라우저 코드·Git에 넣지 않는다.
- Gemini의 기사나 `stockImpactScore`를 실제 역사적 종가라고 표시하거나 실제 종가를 직접 덮어쓰지 않는다.
- 현재 게임 연도 이후의 사건·제품·CEO 교체·실적 결과를 프롬프트나 기사에서 미리 노출하지 않는다.
- AI 연결 실패 때문에 신문이나 다음 날 진행을 막지 않는다. 항상 결정론적 신문 폴백을 유지한다.
- 클라이언트가 보낸 문자열을 추가 시스템 명령으로 실행하거나 프롬프트 지시로 신뢰하지 않는다.
