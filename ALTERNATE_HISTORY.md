# 대체역사 시스템 기준

이 문서는 대체역사 기능의 영구 기준입니다. 전체 현재 상태는 `PROJECT_GUIDE.md`, 바로 다음 작업은 `HANDOFF.md`, 이벤트 작성법은 `CONTENT_GUIDE.md`, 수치 근거는 `BALANCE_NOTES.md`를 봅니다.

## 핵심 경험

플레이어가 실제 회사에 영향력이나 지배권을 얻은 뒤 CEO·이사회 안건을 선택하면 제품, 직원, 현금, 매출, 브랜드와 주가가 지연·확률·연쇄 방식으로 달라집니다. 역사적으로 성공한 선택도 게임 조건에 따라 실패할 수 있고, 역사적으로 실패한 길도 성공할 수 있어야 합니다.

## 절대 규칙

1. 지배권 확보 전에는 역사 데이터 기준선을 사용합니다.
2. 지배권이 생긴 날의 시뮬레이션 가격은 그날의 역사 기준값과 같아야 합니다.
3. 분기 이후에는 실제 미래 가격을 사용하지 않고 게임 상태와 제한된 난수로 계산합니다.
4. 한 번 분기된 회사는 지배권을 잃어도 실제 역사로 되돌아가지 않습니다.
5. 실제 역사 이벤트는 날짜만으로 강제하지 않고 현금, 기술, 인력, CEO, 이사회와 앞선 선택 조건을 검사합니다.
6. 미래의 실제 성공 결과와 내부정보를 미리 보여주지 않습니다.
7. 실제 회사명은 사용할 수 있지만, 공개 자료가 아닌 내부 대화·수치·코드명·결과는 반드시 게임용 가상 시나리오라고 표시합니다.
8. 선택에는 즉시 장점과 비용, 지연 결과와 후속 위험이 함께 있어야 합니다.
9. 모든 난수는 `simulationSeed`와 사건 키에서 결정하며 같은 저장과 같은 선택은 같은 결과를 냅니다.
10. 결정, 지연효과와 원장 항목은 고유 ID를 사용해 두 번 반영되지 않게 합니다.

## 현재 구현된 첫 세로 슬라이스

Flutter의 Apple 대체역사 체험 시나리오에 다음 흐름이 구현되어 있습니다.

```text
경영권 시나리오 선택
→ Project Atlas 제품 제안
→ 투자 규모 선택
→ 배터리 품질 문제
→ 비용·기능·일정 선택
→ 출시 또는 연기·취소
→ 지연된 초기 결과
→ 현금·매출·브랜드·사기·시뮬레이션 기준지수 변화
```

- 실제 Apple 시세나 내부자료가 아니라 개발용 역사 기준지수와 게임용 조건을 사용합니다.
- `worldMode`, `divergedAtDay`, 분기 원인과 분기 가격을 저장합니다.
- 중요한 안건이 있으면 시간 진행 버튼이 안건 열기로 바뀌어 자동정지합니다.
- 출시 연기는 품질과 위험을 개선하지만 현금과 시장 적합도를 희생합니다.
- 승인, 단계 투자, 공동개발, 거절, 품질 보강, 기능 축소, 개발 연장, 출시, 출시 연기와 취소가 서로 다른 결과를 만듭니다.

## Flutter 상태와 처리 순서

- 현재 저장 스키마: `version: 8`
- 논리 저장 키: `simul-millennium-capital-v1`
- 상태: `flutter_app/lib/game/game_state.dart`
- 규칙과 결정론: `flutter_app/lib/game/game_engine.dart`
- 저장, 이전 Flutter 저장과 React 웹 v3 마이그레이션: `flutter_app/lib/game/game_persistence.dart`

플레이 시간은 08:00~20:00의 공통 시계로 진행하고, 오늘 신문 확인 뒤 다음 날짜 08:00으로 넘어갑니다. 날짜가 바뀔 때 분기 기업 가격 계산, 월간 개발비, 만기 지연 이벤트, 안건 자동정지와 저장을 처리합니다. Flutter v13은 실제 시장 거래·포지션·실현손익·AUM을 저장하고, Apple 분기 시점의 실제 가격을 기준으로 대체 가격을 연속적으로 계산합니다. 2010년 마지막 날에는 더 진행하지 않고 최종 결산으로 전환합니다.

## 다음 구현 순서

1. 실제 역사 기준선에서 완전 대체 가격으로 이어지는 Apple 검증
