# 게임플레이 완성 상태

최종 갱신: 2026-07-22

대상은 Flutter 주 구현(`flutter_app/`)입니다. 현재 게임 저장 스키마는 `v13`, 시장 데이터 스키마는 `4`입니다. 이 문서는 과거 결함 목록이 아니라 현재 동작과 회귀 방지 기준만 기록합니다.

## 현재 진행 구조

1. 새 게임은 `2000-01-02 일요일 08:00`, 현금 `0원`, 창립 인원 `1명`으로 시작합니다. 첫 `하루 보내기` 뒤에는 `2000-01-03 월요일 08:00`이 되어 바로 시장을 이용할 수 있습니다.
2. 일거리로 누적 `10,000원`을 직접 벌면 첫 주문 권한이 열립니다. 현재 현금을 써도 누적 진행도는 줄지 않습니다.
3. 보호자 명의 계좌의 권한 레벨이 실제 주문 한도 검사에 연결됩니다. 첫 거래와 신뢰·평판 성장으로 권한이 올라갑니다.
4. 거실 편지함은 `미션·안건 보드`로 동작합니다. 첫 조사노트부터 매수·분산·매도·30일 수익·채용·펀드·부동산·2010년 최종 도전까지 43개 주 미션이 순서대로 열립니다.
5. 모든 안건 결정은 즉시 `25 XP`를 주고 미션 진행을 올립니다. 미션 보상은 XP·게임 현금·평판·가족 신뢰로 구성되며 저장 성공 뒤에만 수령 처리합니다.
6. 경험치 레벨 1~10은 일거리 보상, 거래비용, 월 리서치 수입, 수익 매도 평판, 가족 브리핑, 채용 계약금, 예수금 이자, 부동산 수입, 미션 현금 보상에 실제 패시브 효과를 적용합니다.
7. 시장 화면을 열어 둔 동안만 현실 `1초`마다 게임 시각과 기본 차트가 `1분` 진행합니다. 시장 화면을 닫으면 타이머도 멈춥니다.
8. 하루, 다음 거래일, 1주, 1개월, 다음 결정까지 빠르게 진행할 수 있습니다. 내부 처리는 항상 하루씩 순서대로 실행되어 기업행동·월 결산·결정이 누락되지 않습니다.
9. 2001·2003·2004·2006·2008·2010의 주요 사건과 조직 마일스톤에서 선택 카드가 열리며, 대기 결정이 있으면 자동 진행이 멈춥니다.
10. 2003년부터 후보를 채용하고 월 급여를 지급합니다. 직원은 월 리서치 수입을 만들며, 사무실 단계에 따라 임대료가 달라집니다.
11. 평판과 직원 조건을 충족하면 외부 펀드를 열 수 있고, 외부 AUM과 월 운용보수가 저장됩니다.
12. 작은방 서류함의 `자산·소비 계획`에서 가족 활동, 교육, 직원 연수, 기업자료, 부동산, 장학사업에 실제 게임 현금을 지출합니다. 부동산 월 수입·유지비와 매각 기록은 현금 원장에 남습니다.
13. 2010년 캠페인 마지막 날에는 더 진행하지 않고 최종 결산으로 전환합니다. 현금·포트폴리오 원가·실현손익·수수료·평판·신뢰·조직·외부 AUM·부동산·선택지출·주요 선택을 보여 줍니다.

## 미션·경험치·스킬

- 미션 진행도는 실제 매수·매도·수익 거래·일거리·안건·날짜·가족 도움·월 결산·채용·펀드·소비·부동산·확률 오락 상태에서 계산합니다.
- 상대 목표는 활성화 시점의 카운터를 기준으로 하므로 이전 행동을 새 목표에 중복 계산하지 않습니다. 누적 첫 행동 목표는 이전 저장의 원장과 상태를 안전하게 인식합니다.
- 30일 안에 현금 늘리기는 실패해도 벌점이나 초기화 없이 새 30일 구간으로 자동 재시작합니다. 미션은 시간 진행을 잠그지 않으며 실제 안건만 기존 규칙대로 진행을 멈춥니다.

## 주식 시스템

- 화면, 주문창, 체결 엔진은 같은 권위 시세 서비스와 같은 시장 시각을 사용합니다.
- 매수·매도 수수료, 평균단가, 부분 매도 원가, 실현손익, 원화 AUM을 원장에 저장합니다.
- 분할과 현금배당은 날짜 진행 중 한 번만 반영하고 처리 이벤트 ID를 저장합니다.
- 환율 원장이 없는 해외 자산은 신규 주문을 막습니다. 이전 저장의 거래 불가능한 해외 포지션은 원가를 원화 현금으로 한 번 복구합니다.
- 손상 또는 불완전한 대체역사 가격은 결정론적 역사 가격으로 복구하여 진행 중 크래시를 막습니다.
- 역사 종가는 원본으로 유지하고 장중 1분봉만 결정론적으로 생성합니다.
- 시장 상단에는 현재 주 미션 진행도와 수령 가능한 보상을 표시합니다. 미션 보상 저장 중에는 중복 수령을 막습니다.
- 관심 종목은 별표와 별도 탭으로 관리하고, 종목별 기업 조사노트는 300자까지 저장·수정·삭제하여 재접속 뒤에도 복원합니다.

## 자산·소비 시스템

- 가족 나들이는 월 1회, 투자·회계 서적과 직원 연수와 장학사업은 연 1회로 제한합니다. 기업자료 아카이브는 1회 구매 후 월 리서치 수입을 늘립니다.
- 자가 사무실은 2006년 법인 설립 뒤 구입할 수 있으며, 구입하면 사무실 임대료가 없어지고 월 유지비가 발생합니다.
- 소형 상가 지분은 2008년부터 월 임대수입과 유지비가 각각 결산됩니다. 가족 주택 신탁은 2009년부터 열리고 가족 신뢰와 월 관리비를 함께 반영합니다.
- 부동산은 취득 후 30일이 지나야 매각할 수 있습니다. 매각가는 보유기간과 거래비용을 반영한 결정론적 게임 수치이며 실제 역사 가격이 아닙니다.
- 확률 오락은 2010년 성인 시점에만 열립니다. 월 1회, 최소 1만원, 보유 현금의 1%와 10만원 중 작은 값을 상한으로 사용합니다.
- 지급 확률은 `60%: 0배`, `30%: 1.5배`, `10%: 3배`, 평균 지급률 `75%`로 항상 공개합니다. 실제 결제·광고·현금 보상은 없고 게임머니만 사용합니다.
- 모든 지출·매각·확률 결과는 확인 대화상자를 거쳐 저장 성공 뒤에만 화면에 게시합니다.

## 뉴스·달력·서사

- 거래일 여부는 공용 시장 달력을 사용합니다. 주말, 고정 휴일, 실제 종가 공백에서 추출한 휴장일을 동일하게 처리합니다.
- 같은 날짜의 복수 역사 사건을 한 건만 버리지 않고 대표 기사와 다른 헤드라인으로 함께 표시합니다.
- 동적 기사는 거래일의 주요 사건이나 의미 있는 플레이어 행동이 있을 때만 요청합니다. 실패 시 고정 기사로 즉시 폴백합니다.
- 최근 신문은 최대 90일 링버퍼로 저장하고, 책상 서류함의 장부에서 최근 10일 기사를 다시 볼 수 있습니다.
- 24단계 가족 프롤로그의 시작 선택과 성향은 초기 가족 관계·신뢰·학교 균형에 반영되고, 가족 도움은 조사 보너스·수수료 할인·관계 변화에 연결됩니다.
- 현재 대체역사 캠페인의 중심 기업은 Apple과 Project Atlas입니다. Microsoft·삼성전자·Sony 등은 실제 시장·경영진 조사 대상으로 제공되며, 사실 기록과 게임용 가상 결과를 화면에서 구분합니다.

## 저장·복구

- 상태 스키마 `v13`은 0원 시작, 누적 종잣돈, 주문 권한, 평판, 미션 진행도, 경험치, 레벨, 스킬, 사무실 단계, 펀드, 성과 이력, 뉴스 아카이브, 선택지출, 부동산, 확률 오락 월 제한을 포함합니다.
- 앱 첫 화면은 `처음하기`·`이어하기`이며, 로컬 저장은 최대 5슬롯입니다. 처음하기는 가장 앞의 빈 슬롯을 사용하고 5개가 모두 차면 삭제 화면으로 안내합니다.
- 이어하기에는 회사명·게임 날짜·DAY·레벨·현금·최근 저장 시각을 표시합니다. 각 슬롯은 확인 후 삭제할 수 있으며 기존 단일 저장 키는 1번 슬롯으로 그대로 인식합니다.
- 날짜 진행은 일괄 진행을 포함해 하루 계산이 끝날 때마다 활성 슬롯에 자동저장합니다. 허브 상단 메뉴에서는 현재 슬롯에 즉시 수동저장하고 타이틀로 돌아갈 수 있습니다.
- 이전 Flutter 저장과 React 웹 `v3`의 지원 필드는 회사명·날짜·현금·조직·포지션·원장을 보존해 가져옵니다.
- 마이그레이션은 자본을 임의로 지급하지 않으며 여러 번 실행해도 같은 결과입니다.
- 정상 저장은 슬롯별 백업 1개를 유지합니다. 기본 저장이 손상되면 같은 슬롯의 백업을 복구하고, 둘 다 손상되면 원문을 보존하며 이어하기에서 해당 슬롯을 삭제할 수 있습니다.
- 저장 성공 전에는 미래 상태를 화면에 게시하지 않습니다. 실패하면 기존 상태를 유지하고 재시도할 수 있습니다.

## 모바일·성능·보안

- 기준 뷰포트는 `390×844`, 최소 폭은 `360px`, 앱 프레임 최대 폭은 `430px`입니다.
- 모바일 Web 문서와 Flutter 뷰는 페이지 스크롤을 고정합니다. 이름 입력 중에는 배경·상단 장면표시를 고정하고 이름 패널만 320px 키보드 바로 위로 한 번 이동합니다.
- 주요 뒤로가기와 행동 버튼은 최소 `44×44` 터치 영역을 사용합니다.
- 첫 허브 진입에는 한 번만 보이는 조작 안내와 상시 도움말이 있습니다. 세 방은 원하는 목적지로 바로 이동할 수 있습니다.
- 시장의 주기적 갱신은 현실 1초마다 한 번이고 그때 게임 1분이 진행되며, 화면 이탈·주문·저장 중에는 정지합니다.
- 뉴스 API는 Gemini 3.6 Flash, MEDIUM 추론, Origin 검사, 입력 길이 제한, IP 단위 분당 요청 제한, 구조화 출력 검증과 프롬프트 입력 정리를 적용합니다.

## 검증 기준

- `flutter analyze`
- `flutter test`
- `npm test`
- `npm run build:release`
- 같은 와이파이 모바일에서 `http://<PC-LAN-IP>:8000/play/` 접속

현재 출시 경로를 막는 알려진 P0/P1 게임플레이 결함은 없습니다. 새로운 기업별 대체역사 캠페인과 추가 초상·장소는 기존 규칙을 바꾸는 버그 수정이 아니라 별도 콘텐츠 확장으로 관리합니다.
