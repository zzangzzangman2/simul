# 실제 회사 이벤트 콘텐츠 작성 기준

## 실제 회사 사용 원칙

- Apple, Samsung, Microsoft, Sony처럼 실제 회사명을 사용할 수 있습니다.
- 공개적으로 확인된 사건과 수치는 `DATA_SOURCES.md`에 출처, 공개일과 라이선스를 기록합니다.
- 공개 근거가 없는 CEO 발언, 직원 의견, 예산, 성공확률, 기술점수와 결과는 게임용 가상 시나리오임을 카드와 회사 화면에 표시합니다.
- 당시 플레이 날짜 이후에 공개된 성공 결과, 제품명과 주가를 힌트로 누출하지 않습니다.
- 제품은 공개 전까지 내부 코드명을 사용하고 예상값은 단일 숫자보다 범위와 신뢰도로 보여줍니다.

## 회사당 첫 콘텐츠 묶음

처음부터 수십 개를 만들지 않고 아래 다섯 체인을 우선합니다.

1. 제품 또는 서비스 개발
2. 대규모 설비투자
3. CEO·핵심임원
4. 인수 또는 사업부 매각
5. 위기 대응

각 체인은 `제안 → 예비 투자 → 중간 문제 → 추가 투자/중단 → 출시/완료 심사 → 초기 결과 → 장기 결과`를 기본으로 합니다.

## 결정 카드 필수 필드

- 고유 ID와 회사·프로젝트 ID
- 생성일과 마감일
- 분류와 제안자
- 현재 날짜에 공개 가능한 본문
- 요청 자금
- 핵심 장점과 핵심 위험
- 서로 다른 이해관계를 가진 의견 2~3개
- 잠금 조건과 잠긴 이유
- 즉시 효과, 지연 효과와 후속 카드
- 결과 결정에 사용하는 사건 키

## 선택지 규칙

- 정답처럼 보이는 선택지를 만들지 않습니다.
- 투자 승인이 즉시 매출로 이어지지 않습니다.
- 개발 승인과 출시 승인을 분리합니다.
- 연기는 정보·품질을 늘릴 수 있지만 현금, 선점과 사기에 비용이 있어야 합니다.
- 거절과 취소도 세계를 멈추지 않으며 경쟁사, 직원 이탈 또는 다른 전략으로 이어집니다.
- 현금·이사회·기술·인력 조건을 충족하지 못한 선택지는 숨기지 말고 이유와 함께 잠급니다.

## 첫 Apple 체인

- 회사: Apple
- 내부 코드명: `Project Atlas`
- 주제: 전화·음악·인터넷을 통합한 터치형 휴대기기
- 실제 공개 데이터가 아닌 항목: 시나리오 배정금, CEO/기술자/회계사 의견, 기술·품질·사기·위험 점수, 결과 임계값
- 결과 방향: 큰 성공, 안정적 성공, 저수익 출시, 실패, 취소, 경쟁사 선점

새 카드를 추가할 때는 승인/거절, 지연효과 날짜, 동일 시드 재현과 모바일 긴 문장 레이아웃 테스트를 함께 추가합니다.
