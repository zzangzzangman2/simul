# 부자되기 시뮬레이션

1999년 마지막 밤의 가족 프롤로그를 지나, 열 살 소년이 초기자본 0원과 창립 인원 1명으로 시작하는 2D 모바일 투자회사 경영 시뮬레이션입니다. 가족과 회사를 조사하며 실제 기업의 역사를 따라가거나 바꿉니다.

<p align="center">
  <img src="./public/og-apartment-v2.png" alt="2000년 세로형 3공간 아파트 허브" width="760" />
</p>

## 현재 핵심 플레이 루프

1. 세로형 아파트의 작은방·거실·부엌을 이동하며 실제 물건을 눌러 기능을 엽니다.
2. 작은방 CRT에서 시장을 보고 서류함에서 고급 포트폴리오·현금원장을 확인합니다.
3. 거실에서 안건과 가족·조직을 관리하고, 부엌 전화로 일거리를 선택합니다.
4. `1시간 보내기` 또는 신문 확인을 거치는 `하루 보내기`로 시간을 진행합니다.
5. 실제 기업·경영진·역사 사건을 만나되 선택에 따라 게임 속 대체역사를 만듭니다.

현재 주 구현은 `flutter_app`이며 Android와 Web을 대상으로 합니다. 저장소 루트의 React 앱은 시장·거래 로직과 이전 저장 데이터의 기준 구현으로 보존합니다.

## 현재 플레이 가능 범위

- `처음하기`·`이어하기` 타이틀, 최대 5개 로컬 저장 슬롯, 이어하기 안의 삭제, 하루별 자동저장과 허브 상단 메뉴의 수동저장
- 1999년 12월 31일의 24단계 가족 프롤로그 이후 2000년 1월 2일 일요일 08:00 시작, 첫 하루 진행 뒤 1월 3일 월요일 08:00 시장 이용
- 초기자본 0원, 창립 인원 1명으로 작은방에서 시작하는 투자회사 이야기
- `bg_bedroom_premium_2000.png`, `bg_living_room_premium_2000.png`, `bg_kitchen_premium_2000.png`를 사용하는 세로형 3공간 아파트 허브
- 설거지·문방구 분류·벼룩장터 계산 미니게임과 하루 3회 일거리 제한
- 작은방 CRT·서류함에서 주식시장과 가죽 장부형 고급 포트폴리오·현금원장으로 진입
- 거실 안건 편지·가족 공간에서 안건함과 가족·조직으로, 부엌 유선전화에서 일거리로 진입
- 배경에 그려진 오브젝트와 같은 위치에 놓인 48px 이상의 터치 핫스폿
- 미해결 안건이 없을 때 사용할 수 있는 독립된 `1시간 보내기`·`하루 보내기` 진행 버튼
- 첫 조사노트부터 매수·매도·30일 수익·채용·펀드·부동산까지 이어지는 43단계 미션과 XP·현금·평판·가족 신뢰 보상
- 레벨 1~10에서 거래비용·일거리·리서치·채용·예수금·부동산·미션 보상을 바꾸는 패시브 스킬 10종
- 입력한 투자연구소 이름을 방 상단·시장 계좌·사람들·서류함에 그대로 표시
- 모바일 키보드가 열린 동안 브라우저가 Flutter 전체 화면을 밀어 올리지 않고 프롤로그 대사·이름 입력·완료 버튼을 유지하는 고정 호스트 입력 레이아웃
- 국내 기본 팩 22개 투자와 기존 해외 6개 종목의 별도 참고 탭 보존. 해외 주문은 실제 환율 원장을 연결하기 전까지 잠급니다.
- 토스증권형 홈·주식 탐색·내 계좌, 회사 통장↔증권 예수금 입출금, 실제 일별 종가에 수렴하는 장중 체결가 기반 매수·매도와 기본 증권 수수료, 처분원가·실현손익 원장, 현금·포지션·포트폴리오·운용자산 계산
- 주문 저장 직전 원장에서 종목·날짜·분·가격을 재검증하고, 분할·주식배당·원화 현금배당을 날짜 진행 시 멱등 적용
- 가족 조언자 배치와 피로도·관계·조사 도움, 2003년 첫 조사원 해금 안내
- 닷컴 버블, 9·11, iPod, Google IPO, iPhone, 금융위기 등 역사 이벤트
- 날짜별 실제 역사 소식 400여 건과 사건이 없는 날의 일상 브리핑
- 실제 경영진 12명의 시대별 재임 정보와 카툰 상반신 이미지
- 최대 5개 브라우저 로컬 슬롯, 슬롯별 직전 정상 백업과 손상 원문 보존·자동 복구, 하루별 자동저장·수동저장, 주식시장의 화면·시스템 Back 종료 전 현재 시각 저장, 기존 React v3의 회사명·날짜·현금·포지션을 1번 슬롯으로 가져오기
- 이전 Flutter 저장과 React v3 지원 필드의 현금·날짜·조직·포지션·원장을 보존하는 v13 마이그레이션

화면 가격은 거래일의 실제 일별 종가를 사용합니다. 주식시장 화면이 열려 있는 동안에만 현실 1초마다 게임 시각과 생성 가격이 1분씩 진행되며 기본 차트는 1분봉입니다. 15:30 종가 앵커와 20:00 마지막 틱은 실제 종가와 일치합니다. 화면을 닫으면 타이머가 제거되고, 주말·휴장일에는 직전 종가를 유지하며 미래 가격을 미리 사용하지 않습니다. 자세한 범위와 상용 데이터 전환 계획은 [DATA_SOURCES.md](./DATA_SOURCES.md)를 참고하세요.

Flutter 기본 경로에 평판, 2003년 채용, 월 급여·임대료·리서치 수입, 외부 펀드와 AUM, 연도별 결정, 2010년 최종 결산이 연결됩니다.

## Flutter 앱 실행

새 주 구현은 [`flutter_app`](./flutter_app)에서 진행합니다. Flutter 안정 버전이 필요합니다.

```bash
cd flutter_app
flutter pub get
flutter run -d chrome
```

검증:

```bash
flutter analyze
flutter test
flutter build web --release
```

자동 검증은 엔진·스토리·미니게임·거래·이전 Flutter 저장→v13 보존 마이그레이션·주식시장의 시스템 Back 저장·경영진·시장소식과 `360×800`, `390×844` 세로 화면 레이아웃을 포함합니다. 테스트 개수와 통과 여부는 최종 통합 검증 결과를 기준으로 확인합니다.

## 기존 React 앱 실행

Node.js 22.13 이상이 필요합니다.

```bash
npm install
npm run dev
```

기존 앱은 기능 비교와 회귀 검증을 위한 기준 구현으로 보존합니다. 기본 사용자 경로는 Flutter이며, 기존 React v3의 지원 필드는 Flutter v13 가져오기 경로로 보존합니다.

```bash
npm run build
npm test
```

Flutter Web과 Vinext를 함께 배포용으로 빌드할 때는 저장 브리지와 `/play/` 경로까지 검증하는 `npm run build:release`를 사용합니다.

### 같은 Wi-Fi의 실제 휴대폰에서 테스트

먼저 최신 Flutter Web 번들을 만든 뒤 LAN 개발 서버를 엽니다.

```bash
npm run build:flutter-web
npm run dev:lan
```

PC와 휴대폰을 같은 Wi-Fi에 연결하고 휴대폰 브라우저에서 `http://<PC 내부 IP>:8000`으로 접속합니다. 예를 들어 PC의 내부 IP가 `10.0.0.251`이면 `http://10.0.0.251:8000`입니다. Windows 방화벽 메시지가 나오면 공용 네트워크가 아니라 현재 사용 중인 개인 네트워크에서 Node.js의 8000 포트 접근을 허용합니다. 내부 IP는 네트워크가 바뀌면 달라질 수 있으므로 실행할 때 다시 확인합니다.

## 시장 데이터 갱신

```bash
npm run data:fetch
```

이 명령은 국내 22개·해외 6개의 React·Flutter 공용 종가와 기업행동 스키마 4 스냅샷을 다시 생성합니다. KRX 전 종목은 KRX 계정과 라이선스 확인 후 `npm run data:fetch:krx-full`을 사용합니다.

### 장중 시간과 하루 결산

- 하단의 독립된 `1시간 보내기` 버튼으로 최대 20:00까지 60분씩 진행합니다.
- 15:20 장마감 동시호가, 15:30 실제 종가 확정, 15:40~20:00 NXT형 게임 확장장을 재현합니다.
- `하루 보내기`는 기존 `한국경제신문` 게임 특별판 생성·확인 흐름을 거친 뒤 다음 날 08:00으로 이동합니다. 신문의 X·상단 뒤로가기·시스템 뒤로가기 모두 같은 결과이며 당일 20:00에 남지 않습니다. 미해결 안건이 있으면 두 진행 버튼이 모두 잠깁니다.
- 다음 날 신문을 펼치기 전에 Gemini 3.6 Flash가 날짜, 당일 실제 국내 종가 집계, 상승·하락 상위 종목과 그날까지 공개된 역사 흐름을 바탕으로 경제 기사를 생성합니다. 거래·휴장 여부와 관계없이 매일 생성을 시도하며, 플레이어·가족·투자소·개인 매매는 기사 소재에서 제외합니다. 연결 실패 시 같은 시장 자료로 만든 결정론적 경제신문으로 자동 복귀합니다.
