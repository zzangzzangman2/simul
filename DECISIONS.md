## D-014 — 국내 시장을 기본 무대로 하고 해외 종목은 별도 탭에 보존

- 상태: 채택
- 결정: 주가창 기본 화면은 국내 22개 개발용 팩으로 시작하고 KOSPI·KOSDAQ 필터를 제공하며, 기존 미국·일본 6개 종목은 `해외` 탭에 보존합니다.
- 이유: 국내 기업을 골라 성장·인수하는 핵심 재미에 집중하면서 이미 만든 해외 데이터와 향후 해외 진출 확장성을 잃지 않기 위해서입니다.
- 가격 규칙: 개입 전 거래일 마감값은 실제 일별 종가를 사용합니다. 시장 화면이 열려 있을 때만 현실 1초당 게임 1분이 진행하며 장중 가격은 실제 종가로 수렴합니다.
- 틱 움직임: 종목·날짜별 시드의 난수 충격, 단기 모멘텀, 변동성 군집과 드문 급변을 합성한 뒤 브리지 보정으로 실제 종가에 도달시킵니다. 반복 파동은 사용하지 않으며 실제 일봉과 게임용 1분봉을 분리합니다.
- 분봉 차트: 기본과 최소 단위는 1분이며, 실제 체결 분봉이 아닌 결정론적 게임 연출임을 화면에 구분합니다.
- 증권 UX: 토스증권처럼 가격→전일 대비→차트→기간→핵심 지표→사기/팔기 순서로 읽히게 하고, 주문은 10살 주인공 설정에 맞춰 부모 승인 시트로 연결합니다.
- 전 종목 확장: `scripts/fetch-krx-full-market.py`가 모든 평일 상장목록 합집합과 원시 종가를 수집합니다. KRX 계정과 재배포 권한 확인 전에는 생성물을 공개 저장소에 넣지 않습니다.
- 대체된 결정: D-004의 정규화 수정주가만 표시하는 방식은 보조 `adjustedIndex`로 남기고, 화면 가격은 원시 일별 종가로 전환합니다.

## D-015 — Flutter에서 실제 거래와 기존 웹 진행을 계승

- 상태: 대체
- 결정: 기본 사용자 경로인 Flutter에 실제 일별 종가를 앵커로 삼는 현재 장중 게임 체결가 기반 매수·매도, 거래비용, 포지션과 원화 AUM 원장을 연결하고 상태 스키마에 포함합니다. 기존 React 웹 v3 저장은 명시적인 가져오기 마이그레이션으로 계승합니다.
- 이유: 가족 프롤로그 뒤의 조사와 부모 승인 흐름이 실제 투자 결과로 이어지게 하고, 기술 전환 때문에 기존 플레이 진행을 잃지 않기 위해서입니다.
- 당시 결과: React 앱은 회귀 비교용으로 남고 기본 플레이와 저장의 기준을 Flutter로 옮겼습니다. 같은 저장을 중복 변환하거나 이전 진행을 조용히 삭제하면 안 됩니다. 실제 환율 원장이 없는 USD·JPY 주문은 참고 전용으로 잠가 외화 가격을 원화처럼 계산하지 않습니다.
- 대체된 결정: D-016의 Flutter v13 보존 마이그레이션이 현재 저장 기준을 계승합니다.

## D-2026-07-21 동적 뉴스와 결정론적 가격 엔진 분리
